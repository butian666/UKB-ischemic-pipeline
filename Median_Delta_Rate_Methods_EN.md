# Median Delta Rate (SCI-style Methods; aligned with current analysis code)

## Definition and Rationale

Median Delta Rate is a robust, distribution-insensitive summary statistic designed to quantify group-level longitudinal change for imaging-derived phenotypes (IDPs) between two imaging assessments. The metric is constructed at the individual level and then aggregated by group medians to reduce sensitivity to outliers and non-Gaussian change distributions.

## Longitudinal Pairing and Notation

For each participant \(i\), an IDP is paired across the two UK Biobank imaging instances: baseline (Instance.2) and follow-up (Instance.3). Let \(Y_{i1}\) and \(Y_{i2}\) denote the baseline and follow-up measurements of the same IDP, respectively. The absolute change is defined as:

\[
\Delta Y_i = Y_{i2}-Y_{i1}.
\]

IDP pairing is performed using standardized variable-name rules that strip the instance suffix to obtain an IDP root, ensuring that the same phenotype (including array position when present) is matched across \(t_1\) and \(t_2\). Computations are restricted to participants with finite values required by the selected rate definition.

## Individual-level Change Rate

An individual-level change rate \(r_i\) is computed from \(\Delta Y_i\) using a normalization scheme selected a priori or via runtime analysis options.

### Primary (baseline-normalized) rate

To avoid inflated ratios when \(Y_{i1}\) is near zero, the primary implementation uses a group-referenced baseline denominator when a group indicator is available. Let \(g(i)\in\{\mathrm{control},\mathrm{case}\}\) denote the group of participant \(i\), and define the group-specific baseline reference:

\[
d_g=\left|\mathrm{mean}(Y_{i1}\mid g)\right|.
\]

The baseline-normalized rate is then:

\[
r_i=\frac{\Delta Y_i}{d_{g(i)}}.
\]

If \(d_g\) is non-finite or \(\le \epsilon\), the rate is set to missing for affected participants. In the implementation, \(\epsilon\) corresponds to machine precision (i.e., \(\epsilon=\texttt{.Machine\$double.eps}\)).

When a group indicator is not available, the denominator defaults to the individual absolute baseline level with the same near-zero safeguard:

\[
r_i=\frac{\Delta Y_i}{|Y_{i1}|},\quad \text{evaluated only when }|Y_{i1}|>\epsilon.
\]

### Sensitivity rate definitions

Two alternative rate definitions are supported for sensitivity analyses:

1) Symmetric normalization:
\[
r_i^{\mathrm{symmetric}}=\frac{Y_{i2}-Y_{i1}}{\frac{|Y_{i1}|+|Y_{i2}|}{2}},
\]
computed only when the denominator is finite and \(>\epsilon\).

2) Log-ratio change:
\[
r_i^{\mathrm{log\_ratio}}=\log(Y_{i2})-\log(Y_{i1}),
\]
computed only when \(Y_{i1}>0\) and \(Y_{i2}>0\).

## Outlier Mitigation (Optional Winsorization)

To reduce the influence of extreme individual rates, optional winsorization can be applied to \(r_i\). For a winsorization fraction \(f\in(0,0.5)\), finite \(r_i\) values are clipped to the empirical quantiles \(Q_f\) and \(Q_{1-f}\):

\[
r_i \leftarrow \min\left(\max(r_i, Q_f), Q_{1-f}\right).
\]

Quantiles are computed using the standard Type-7 definition.

## Percent Scaling

For interpretability, individual rates are expressed as percentage points:

\[
\mathrm{delta\_percent}_i = 100\cdot r_i.
\]

## Group-level Median Contrast (Median Delta Rate)

Group-wise medians of \(\mathrm{delta\_percent}_i\) are computed separately for cases and controls, and the case–control contrast is defined as:

\[
\Delta_{\mathrm{median}}=
\mathrm{median}(100r_i\mid \mathrm{case})-
\mathrm{median}(100r_i\mid \mathrm{control}).
\]

\(\Delta_{\mathrm{median}}\) is computed independently for each IDP (or IDP root) using participants with finite \(\mathrm{delta\_percent}_i\). Positive values indicate a larger median increase (or smaller median decrease) in cases relative to controls; negative values indicate a larger median decrease (or smaller median increase) in cases relative to controls.

## Reporting and Traceability

Median Delta Rate outputs are generated by the main analysis pipeline as IDP-wise summaries and written to longitudinal change summary tables. The selected rate definition and winsorization behavior are controlled via runtime options used during analysis execution.
